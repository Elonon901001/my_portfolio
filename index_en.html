Project 01

Fine-Tuning Qwen2.5-VL for Leakage-Aware Multimodal Land-Use Classification

Qwen2.5-VL · LoRA / QLoRA · Closed-set Classification

This project targets parcel-level land-use classification in New York City. I built a multimodal benchmark with 84,544 tax lots by aligning NYC PLUTO 2018 records with 2018 orthophoto imagery. Each sample contains a 512×512 aerial image crop, weak administrative metadata, and a text description, and the task is to predict one of 11 fixed land-use codes. The core work was 4-bit LoRA adaptation of Qwen2.5-VL-7B-Instruct under image-only, weak-metadata-only, and image-plus-metadata settings.

Benchmark Construction

Starting from NYC public PLUTO 2018 records and 2018 orthophoto tiles, I cleaned pluto2018.csv, standardized BBL identifiers and land-use labels, sampled 84,544 parcels from 855K valid lots with borough × landuse stratification, and converted NYC coordinates to latitude and longitude.

I then cropped 512×512 parcel-level aerial images from orthophoto tiles, constructed JSONL manifests with image, metadata, and text fields, split train / validation / test sets, and produced both landuse11_full and landuse11_weakmeta dataset versions.

Leakage-aware Design

  • Identified shortcut / leakage risks in PLUTO metadata, including bldgclass, zonedist1, landuse, and building-area fields.
  • Found through rule-based checks that bldgclass could almost directly predict land use, with accuracy and macro-F1 close to 0.9999.
  • Built a weakmeta setting that keeps only borough, landmk, histdist, yearbuilt, lotarea, numfloors, latitude, and longitude.
  • Used weak metadata to avoid shortcut learning from administrative proxy fields and make multimodal evaluation fairer.

Qwen2.5-VL Adaptation

  • Loaded Qwen2.5-VL-7B-Instruct and used 4-bit quantization with LoRA adapters for parameter-efficient fine-tuning.
  • Used HuggingFace Trainer for supervised fine-tuning, saved final_adapter, and evaluated on the held-out test set.
  • LoRA settings included rank 16, alpha 32, dropout 0.05, learning rate 2e-4, and 2 epochs.
  • Trained three modality-specific adapters: image_only, weakmeta_only, and image_weakmeta.

Closed-set Candidate Scoring

To avoid unstable free-form VLM outputs, I implemented closed-set candidate scoring. Instead of allowing the model to generate arbitrary text, the evaluator computes conditional log probabilities for each of the 11 fixed land-use codes and selects the highest-scoring label.

Baselines and Distillation

  • Compared against traditional multimodal baselines including ResNet50 image-only, ResNet50 image+weakmeta, no-gate fusion, and gated fusion.
  • Qwen LoRA image+weakmeta closed-set reached 0.7060 macro-F1, outperforming ResNet50 image+weakmeta at 0.6627 and gated fusion at 0.6716.
  • Cached Qwen closed-set probabilities over 11 classes and explored Qwen-derived soft-label distillation into lightweight ResNet students.
  • Distillation experiments suggested that direct VLM adaptation was more effective than Qwen-to-ResNet student distillation in the current setup.

Project 02

Yanzhi | AI-based Facial Aesthetics Analysis System

Multi-agent · Multimodal · Cloud Deployment · RAG

I developed a multi-agent, multimodal facial aesthetics analysis system for frontal face images. The system integrates computer vision, geometric modeling, LLM-based reasoning, and RAG-constrained scoring into a unified structured evaluation framework. A shared representation layer enables specialized agents to reason over the same visual features, landmarks, geometric descriptors, and texture cues.

Project Overview

The system uses a purpose-built multi-agent architecture. Each agent performs expert-level analysis for a specific facial region, while all agents operate on a unified structured representation layer for shared features, cross-module coordination, and chained reasoning.

I also built a RAG-based scoring framework that anchors model outputs to curated aesthetics rules and skincare/style knowledge bases. This improved output stability, JSON consistency, and robustness across lighting conditions and user image styles.

The full system is deployed on Alibaba Cloud uniCloud as a production-oriented end-to-end pipeline, including a Uni-app frontend built with Vue3 and TypeScript, Node.js cloud-function inference scheduling, asynchronous task queues, content safety checks, cloud persistence, hot updates, and a scalable cloud-native architecture.

Project 03

Thesis: Multimodal LLM-RAG System for Cotton Disease and Pest Diagnosis

Thesis · Qwen-VL · RAG · Milvus · Agricultural AI

This is my thesis project, focused on cotton disease and pest identification, treatment recommendation generation, and field-management question answering. I researched and implemented a multimodal LLM-RAG diagnosis system that combines user-uploaded cotton disease/pest images, natural-language questions, Qwen-VL visual understanding, and retrieved domain knowledge from a cotton plant-protection knowledge base. The system generates structured diagnosis results, supporting evidence, treatment suggestions, and notes. Compared with pure image classification or direct multimodal prompting, the RAG-enhanced design grounds model outputs in agricultural knowledge, reducing hallucination risk and improving the reliability of professional recommendations.

Thesis Goal and System Workflow

The thesis focuses on three practical challenges in agricultural diagnosis: complex field images make pure visual models unstable, general multimodal models can produce recommendations without plant-protection evidence, and diagnosis results need to be structured for farmers and agronomy users.

The system connects image recognition, knowledge retrieval, and LLM generation into an end-to-end diagnosis chain. After a user uploads a field image and enters a question, Qwen-VL extracts visual semantic information and possible disease/pest clues, the retrieval module recalls Top-k professional knowledge snippets, and a prompt template constrains the final answer into diagnosis result, evidence, treatment suggestions, and notes.

Project 04

Multi-Model Sentiment Classification Study with RoBERTa Fine-Tuning

Transformer Encoder · Slurm GPU Cluster · Sentiment Classification

This project compares LSTM, Bi-LSTM+Attention, RoBERTa, and DeepSeek on Yelp, IMDB, and Rotten Tomatoes sentiment classification tasks, covering traditional RNNs, attention-based recurrent models, fine-tuned Transformer encoders, and LLM inference. The overall study shows that vanilla LSTM has a clear bottleneck on fine-grained Yelp 5-class sentiment classification, Bi-LSTM+Attention substantially improves contextual modeling, and Transformer-based methods perform best overall. My main contribution focused on the RoBERTa / BERT advanced Transformer component, including the training pipeline, GPU-cluster execution, strict validation/test refactor, confusion matrices, and experiment artifact archiving.

Project Scope and My Role

The overall project first builds a balanced Yelp 5-class setup and supplements it with IMDB / Rotten Tomatoes binary classification, comparing model behavior from LSTM baselines to Bi-LSTM+Attention, RoBERTa fine-tuning, and DeepSeek.

I focused on adapting pretrained Transformer encoders into a standardized, batch-runnable, auditable training and evaluation system. The general BERT script supports IMDB, Rotten Tomatoes, Yelp binary, Yelp 5-class, Financial PhraseBank, and local CSV inputs, and automatically saves accuracy, macro-F1, confusion matrices, predictions, training time, inference latency, parameter count, and checkpoints.
