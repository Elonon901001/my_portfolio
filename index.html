项目 01

基于 Qwen2.5-VL 微调的纽约地块级土地利用多模态分类

Qwen2.5-VL · LoRA / QLoRA · Closed-set Classification

本项目面向纽约市地块级土地利用分类任务，基于 NYC PLUTO 2018 和 2018 orthophoto imagery 构建包含 84,544 个 tax lots 的多模态 benchmark。每个样本包含 512×512 航拍图、弱行政元数据和文本描述，任务目标是预测固定 11 类 land-use code。 项目重点对 Qwen2.5-VL-7B-Instruct 进行 4-bit LoRA 参数高效微调，并比较 image-only、weakmeta-only 和 image+weakmeta 三种输入设置。

数据集构建

项目从 NYC 官方 PLUTO 2018 表和 2018 航拍图瓦片出发，清洗 pluto2018.csv，标准化 BBL 与 land-use label， 从 85.5 万有效地块中按 borough × landuse 分层抽样 84,544 个样本，并将 NYC 坐标转换为经纬度。

我进一步从 orthophoto tiles 中裁剪 512×512 地块航拍图，构造 image + metadata + text 的 JSONL manifest， 切分 train / validation / test，并形成 landuse11_full 和 landuse11_weakmeta 两个数据版本。

Benchmark 规模

84,544 tax lot samples
11 类 land-use labels
image + metadata multimodal inputs
Split Samples Input
Train 67,635 image + weak metadata + text
Validation 8,453 stratified by borough × landuse
Test 8,456 11-way land-use code

Leakage-aware 设计

  • 识别 PLUTO metadata 中的 shortcut / leakage 风险，例如 bldgclass、zonedist1、landuse 和建筑面积类字段。
  • 通过字段规则检查发现 bldgclass 几乎可以直接预测 land-use，accuracy 和 macro-F1 都接近 0.9999。
  • 构造 weakmeta 版本，只保留 borough、landmk、histdist、yearbuilt、lotarea、numfloors、latitude 和 longitude。
  • 通过弱元数据设置避免模型从行政代理字段中“抄答案”，使多模态评估更加公平。

Qwen2.5-VL 微调

  • 加载 Qwen2.5-VL-7B-Instruct，使用 4-bit quantization 和 LoRA adapter 进行参数高效微调。
  • 使用 HuggingFace Trainer 完成监督微调，保存 final_adapter 并在 test set 上评估。
  • LoRA 设置包括 rank 16、alpha 32、dropout 0.05，学习率 2e-4，训练 2 个 epoch。
  • 分别训练 image_only、weakmeta_only 和 image_weakmeta 三个 modality-specific adapters。

Closed-set Candidate Scoring

为解决 VLM 自由生成标签不稳定的问题，我实现了 closed-set candidate scoring：不让模型自由输出类别文本， 而是对固定 11 个 land-use code 分别计算条件 log probability，并选择概率最高的类别。

Baseline 对比与蒸馏探索

  • 对比 ResNet50 image-only、ResNet50 image+weakmeta、no-gate fusion 和 gated fusion 等传统多模态 baseline。
  • Qwen LoRA image+weakmeta closed-set 达到 0.7060 macro-F1，超过 ResNet50 image+weakmeta 0.6627 和 gated fusion 0.6716。
  • 缓存 Qwen closed-set 的 11 类概率，探索将 Qwen soft label 蒸馏到轻量 ResNet student。
  • 蒸馏结果显示当前设置下直接适配 VLM 优于 Qwen-to-ResNet student distillation。

项目价值

项目 02

颜知 | 基于 AI 的面部美学分析系统

多智能体 · 多模态 · 云端部署 · RAG

我开发了一套面向正脸图像的多智能体、多模态面部美学分析系统，将计算机视觉、几何建模、基于大模型的推理以及 RAG 约束评分整合为统一、结构化的评估框架。系统以共享的结构化表示层为基础，让不同智能体围绕同一组视觉特征、 关键点、几何描述符与纹理线索进行协作分析，实现分层推理与模块化扩展。

项目概述

系统采用专门设计的多智能体架构，每个智能体针对特定部位执行专家级分析。所有智能体基于统一的 “结构化表示层”工作，从而支持共享表征、跨模块协同以及链式推理。

我进一步构建了基于 RAG 的评分框架，将大模型输出锚定到整理后的美学规则库与护肤 / 风格知识库上， 显著提升了输出稳定性、JSON 结构一致性，并降低了不同光照与用户图像风格下的生成波动。

整套系统部署在阿里云 uniCloud 上，形成具备生产可用性的端到端管线，包括：基于 Uni-app （Vue3 + TypeScript）的跨端前端、Node.js 云函数推理调度、异步任务队列、内容安全校验、云端持久化、 热更新发布与可扩展的云原生架构。

项目 03

毕业论文：基于多模态大模型与 RAG 的棉花病虫害智能诊断系统

Thesis · Qwen-VL · RAG · Milvus · Agricultural AI

本项目是我的毕业论文工作，面向棉花种植中的病虫害识别、防治建议和田间管理问答需求，研究并实现了一套基于多模态大模型与 检索增强生成 RAG 的智能诊断系统。系统将棉花病虫害图片、用户问题、Qwen-VL 图像语义理解结果和专业知识库检索结果共同输入大模型， 引导模型生成结构化诊断结果、判断依据、防治建议和注意事项。相比单纯图像分类或直接调用多模态模型，该系统通过农业知识库约束生成过程， 降低大模型幻觉和不专业建议风险，使输出更接近植保实际应用。

论文目标与系统流程

论文关注农业病虫害诊断中的三个问题：田间图像复杂导致单一视觉模型不稳定，通用多模态模型容易给出缺乏植保依据的建议， 以及诊断结果需要以农户和植保人员可读的结构化形式输出。

系统将图像识别、知识检索和大模型生成串联为端到端诊断链路。用户上传田间图片并输入问题后，Qwen-VL 提取图像语义信息和疑似病虫害线索， 检索模块根据识别结果与用户问题召回 Top-k 专业知识片段，最终由 Prompt 模板约束模型输出诊断结果、判断依据、防治建议和注意事项。

知识增强诊断设计

  • 构建棉花病虫害领域知识库，整理棉花常见病虫害的症状特征、发生规律、防治方法和田间管理建议。
  • 对知识文本进行清洗、切片和结构化处理，为后续向量化与检索提供稳定数据基础。
  • 设计知识向量化流程，使用 Embedding 模型将知识片段转为向量并存入 Milvus。
  • 根据用户问题和图像识别结果进行 Top-k 相似知识检索，为多模态推理提供专业上下文。
  • 优化 Prompt 模板和结构化输出格式，要求模型按“诊断结果、判断依据、防治建议、注意事项”组织回答。

  • 重点分析复杂田间图像场景下的泛化能力、解释能力和专业建议可靠性。

我的贡献

  • 提出并实现多模态 RAG 诊断链路，将图像、用户问题、识别结果和检索知识统一输入 Qwen-VL。
  • 搭建棉花病虫害领域知识库和 Milvus 向量检索流程，实现农业知识片段的语义召回。
  • 开发 Web 原型系统，实现图片上传、文本提问、后端诊断调用、知识检索和结果展示闭环。
  • 优化农业诊断 Prompt 与结构化输出，提高系统在植保场景中的可用性、可解释性和输出稳定性。

通过 RAG 引入专业知识库，降低大模型凭空生成和不准确防治建议的风险，为毕业论文提供明确研究问题、系统实现和评估路径。
项目 04

多模型情感分类对比实验：RoBERTa Fine-tuning 与 Transformer 评估管线

Transformer Encoder · Slurm GPU Cluster · Sentiment Classification

本项目比较 LSTM、Bi-LSTM+Attention、RoBERTa 和 DeepSeek 在 Yelp、IMDB、Rotten Tomatoes 情感分类任务上的表现， 覆盖传统 RNN、attention-based RNN、fine-tuned Transformer encoder 和 LLM 推理方法。项目整体结论显示，普通 LSTM 在细粒度 Yelp 5-class 任务上存在明显性能瓶颈，Bi-LSTM+Attention 显著提升上下文建模能力，而 Transformer-based models 表现最强。 我的主要工作集中在 RoBERTa / BERT advanced Transformer 部分，包括训练管线、GPU cluster 运行、strict validation/test 改造、 混淆矩阵和实验产物归档。

项目范围与我的职责

整体项目先构建平衡 Yelp 5-class 数据集，并在 IMDB / Rotten Tomatoes 二分类任务上补充评估，对比从 LSTM baseline 到 Bi-LSTM+Attention、RoBERTa fine-tuning 和 DeepSeek 的性能差异。

我负责把预训练 Transformer encoder 适配成标准化、可批量运行、可复核的训练与评估系统。通用脚本支持 IMDB、 Rotten Tomatoes、Yelp binary、Yelp 5-class、Financial PhraseBank 和本地 CSV，并自动保存 accuracy、macro-F1、 confusion matrix、predictions、training time、inference latency、parameter count 与模型 checkpoint。

VGG19 Transfer Learning 图像分类

基于 Dogs vs. Cats 数据集完成一个端到端视觉分类工程，用 ImageNet 预训练 VGG19 做特征提取，只训练最后的分类器完成 cat / dog 二分类。

  • 完成 224×224 resize、normalization、horizontal flip、small rotation 和 random resized crop 等预处理与增强。
  • 使用 CrossEntropyLoss + SGD 训练 10 个 epoch，并通过 dropout 缓解小数据集过拟合。
  • 使用最佳权重预测 500 张 test set 图片并生成 submission.csv，validation accuracy 约 98% - 99%。
  • 扩展到 CIFAR-10 十分类，并讨论 class imbalance 下的 weighted loss 与 oversampling。

PCA Fashion-MNIST 降维分析

围绕 Fashion-MNIST 构建传统机器学习实验，研究 PCA 降维对 1-NN 图像分类性能的影响。

  • 处理 60,000 张训练图像和 10,000 张测试图像，将 28×28 灰度图展开为 784 维向量。
  • 使用 train-only standardization 防止测试集泄漏，并建立 full-dimensional 1-NN baseline。
  • 系统评估 5、10、20、30、50、80、100、150、200、300、500、784 维 PCA 设置。
  • 300 维 PCA 达到 84.78% 测试准确率，高于 84.13% baseline，并可视化解释方差与准确率曲线。

Code-Review Agent

构建一个轻量级多智能体代码评审原型，重点展示 LLM 应用编排、状态流转和结构化报告生成能力。

  • 前端使用 React，后端通过兼容 LangChain 的 Qwen 模型包装层调用大模型。
  • 设计共享状态机，组织思考、执行、观察、决策和结果聚合五类节点。
  • 引入“评审生成 - 评审验证”的迭代闭环，让系统先生成问题，再检查报告质量并修正。
  • 输出结构化代码评审报告，为后续接入真实 repo、静态分析工具和 PR workflow 预留扩展空间。
